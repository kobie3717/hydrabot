Installation Guide

HydraBot Stack

Multi-agent AI infrastructure powered by Claude Code CLI
Version 1.0
Platform Ubuntu 22.04 / Debian 12
Requires Claude Max subscription

0 Overview

HydraBot is a deployable AI agent stack. Each bot is a Telegram interface powered by Claude Code CLI, connected to a shared memory system (AI-IQ) and a multi-agent mesh (Circus). Ephemeral sub-workers handle background tasks in parallel.

Telegram API │ ▼ PM2 Bots (grammy + Claude Code CLI) │ ├─── circus-bridge.mjs ──► Circus API :6200 (FastAPI, systemd) │ │ │ ▼ │ AI-IQ memory (SQLite + vectors) │ └─── bot-circus/dispatch.mjs ──► Ephemeral Claude workers (up to 10 parallel)
LayerComponentPurpose
0Claude Code CLIAI inference engine — all bots call this
1AI-IQPer-agent long-term memory
2CircusAgent mesh: registry, rooms, task queue
3Bot-CircusEphemeral worker pool for sub-tasks
4circus-bridge.mjsNode.js ↔ Circus integration module
5BotsTelegram interface + business logic

2 Clone the Repository

# Clone anywhere you like (e.g. your home directory)
git clone <repo-url> ~/hydrabot
cd ~/hydrabot
ℹ️
You can install to any path. The installer auto-detects its location. You can also override with the HYDRABOT_DIR environment variable.

3 Run the Installer

# Run the installer (uses sudo only for system packages)
bash ~/hydrabot/deploy/install.sh

The installer performs these steps automatically:

  • Installs system packages: curl git python3 python3-pip sqlite3 build-essential
  • Installs Node.js 22 via NodeSource (skips if already installed)
  • Installs PM2 globally and configures systemd startup
  • Installs Claude Code CLI via npm (skips if already installed)
  • Installs Python packages: ai-iq circus-agent
  • Bootstraps the Circus data directory and generates a keypair
  • Installs bot-circus dependencies and initialises performer workspaces
  • Installs and starts the Circus API as a systemd service
⚠️
Auth pauseIf Claude Code CLI is not already authenticated, the installer will stop and print:

Claude Code CLI installed but needs authentication.
Run 'claude' interactively to authenticate, then re-run this script.

Open a new terminal, run claude, complete the browser login with your Claude Max account, then re-run the installer command.

Verify installation

# Circus API should be healthy
curl http://localhost:6200/health
# → {"status": "ok"}

# Claude CLI should respond
claude -p "say hello"

# PM2 should be running
pm2 list

4 Configure Your First Bot

Get a Telegram bot token

  • Open Telegram and search for @BotFather
  • Send /newbot and follow the prompts
  • Copy the token — it looks like 7123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Get your Telegram user ID

  • Search for @userinfobot in Telegram
  • Send it any message — it replies with your numeric ID

Create and configure the bot

# Copy the template
cp -r ~/hydrabot/bots/template ~/hydrabot/bots/mybot

# Copy and edit the env file
cp ~/hydrabot/bots/template/.env.example ~/hydrabot/bots/mybot/.env
nano ~/hydrabot/bots/mybot/.env

Required .env values

VariableRequiredDescription
TELEGRAM_BOT_TOKENrequiredToken from @BotFather
ALLOWED_USER_IDrequiredYour numeric Telegram user ID (from @userinfobot)
BOT_NAMEoptionalDisplay name for the bot — default: MyBot
CLAUDE_CLI_PATHoptionalPath to claude binary. Default: claude (uses PATH). Get with which claude
CLAUDE_WORKING_DIRoptionalWorking directory for Claude. Default: current directory
CLAUDE_TIMEOUToptionalResponse timeout in ms. Default: 120000 (2 min)

Start the bot

# Install dependencies
npm install --prefix ~/hydrabot/bots/mybot

# Start with PM2
pm2 start ~/hydrabot/bots/mybot/bot.mjs --name mybot

# Save so it survives reboot
pm2 save

# Watch logs
pm2 logs mybot
✅
Bot is runningOpen Telegram, find your bot, and send it a message. It should respond via Claude within a few seconds.

5 Autonomous Worker (Optional)

The worker polls a job queue and executes long-running tasks via Claude. Useful for background operations triggered from Telegram (/task style commands).

# Configure
cp ~/hydrabot/performers/worker/.env.example \
   ~/hydrabot/performers/worker/.env
nano ~/hydrabot/performers/worker/.env

# Install deps
npm install --prefix ~/hydrabot/performers/worker

# Start
pm2 start ~/hydrabot/performers/worker/worker.mjs --name worker
pm2 save
VariableRequiredDescription
TELEGRAM_BOT_TOKENoptionalIf set, worker sends job completion messages via Telegram
TELEGRAM_CHAT_IDoptionalYour chat ID — where job results are sent
CLAUDE_CLI_PATHoptionalPath to claude binary
CLAUDE_WORKING_DIRoptionalDirectory Claude operates in during jobs

6 Running Multiple Bots

Each bot is an independent directory. Repeat the configure step for each one:

# Second bot
cp -r ~/hydrabot/bots/template ~/hydrabot/bots/designbot
cp ~/hydrabot/bots/template/.env.example ~/hydrabot/bots/designbot/.env
# Edit .env with different TELEGRAM_BOT_TOKEN
npm install --prefix ~/hydrabot/bots/designbot
pm2 start ~/hydrabot/bots/designbot/bot.mjs --name designbot
pm2 save
ℹ️
All bots share one Claude Code CLI process pool and one Circus mesh. Each bot gets its own Telegram token, allowed user, and performer workspace.

7 Troubleshooting

SymptomCauseFix
Bot doesn't respond Claude not authenticated Run claude -p "hello" manually. If it fails, run claude login
claude: command not found CLI not in PATH Run which claude — if empty, run npm install -g @anthropic-ai/claude-code. Then set CLAUDE_CLI_PATH in .env
Circus health check fails Service not running systemctl status circus-api — if failed, check journalctl -u circus-api -n 50
Bot starts then crashes Missing env var pm2 logs mybot — look for "is required" errors. Check .env has TELEGRAM_BOT_TOKEN and ALLOWED_USER_ID
Auth works locally but not on VPS SSH session has no browser Run claude login — it prints a URL. Open that URL in any browser to complete OAuth
Circus not connecting Wrong URL Default is http://127.0.0.1:6200. Set CIRCUS_URL in .env if Circus runs elsewhere

Useful commands

pm2 list                    # all running processes
pm2 logs mybot              # live bot logs
pm2 restart mybot           # restart a bot
pm2 stop mybot              # stop a bot
systemctl status circus-api # Circus API status
curl http://localhost:6200/health # quick Circus health check
claude -p "hello"          # test Claude CLI directly
